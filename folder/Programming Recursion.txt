<div>Programming recursion:</div><div><br></div><div>A function that calls itself is known as recursive function and the process of calling function itself is known as recursion in C programming.</div><div>Example of recursion in C programming</div><div>Write a C program to find sum of first n natural numbers using recursion. Note: Positive integers are known as natural number i.e. 1, 2, 3....n</div><div><br></div><div>#include &lt;stdio.h&gt;</div><div>int sum(int n);</div><div>int main(){</div><div>&nbsp; &nbsp; int num,add;</div><div>&nbsp; &nbsp; printf("Enter a positive integer:\n");</div><div>&nbsp; &nbsp; scanf("%d",&amp;num);</div><div>&nbsp; &nbsp; add=sum(num);</div><div>&nbsp; &nbsp; printf("sum=%d",add);</div><div>}</div><div>int sum(int n){</div><div>&nbsp; &nbsp; if(n==0)</div><div>&nbsp; &nbsp; &nbsp; &nbsp;return n;</div><div>&nbsp; &nbsp; else</div><div>&nbsp; &nbsp; &nbsp; &nbsp;return n+sum(n-1); &nbsp; &nbsp;/*self call &nbsp;to function sum() */</div><div>}</div><div><br></div><div>Output</div><div>------</div><div>Enter a positive integer:</div><div>5</div><div>15</div><div><br></div><div>In, this simple C program, sum() function is invoked from the same function. If n is not equal to 0 then, the function calls itself passing argument 1 less than the previous argument it was called with. Suppose, n is 5 initially. Then, during next function calls, 4 is passed to function and the value of argument decreases by 1 in each recursive call. When, n becomes equal to 0, the value of n is returned which is the sum numbers from 5 to 1.</div><div><br></div><div>For better visualization of recursion in this example:</div><div><br></div><div>sum(5)</div><div>=5+sum(4)</div><div>=5+4+sum(3)</div><div>=5+4+3+sum(2)</div><div>=5+4+3+2+sum(1)</div><div>=5+4+3+2+1+sum(0)</div><div>=5+4+3+2+1+0</div><div>=5+4+3+2+1</div><div>=5+4+3+3</div><div>=5+4+6</div><div>=5+10</div><div>=15</div><div><br></div><div>Every recursive function must be provided with a way to end the recursion. In this example when, n is equal to 0, there is no recursive call and recursion ends.</div><div><br></div><div>Advantages and Disadvantages of Recursion</div><div>-----------------------------------------</div><div><br></div><div>Recursion is more elegant and requires few variables which make program clean. Recursion can be used to replace complex nesting code by dividing the problem into same problem of its sub-type.</div><div><br></div><div>In other hand, it is hard to think the logic of a recursive function. It is also difficult to debug the code containing recursion.</div>
